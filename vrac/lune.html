1. La page BROL 2.0
C’est la page que tu vois sur https://rg-ashy.vercel.app/. Elle sert de tableau de bord : une liste de modules à gauche, un grand “écran” au centre pour afficher les pages, et des “embeds” (YouTube, PDF, etc.).
dashboard navigation
2. La page “GR”
C’est la page sur https://rg-ashy.vercel.app/gr qui présente un mouvement d’éducation permanente citoyenne, avec une partie vitrine et une partie intranet / console.
éducation permanente intranet
Idée clé (niveau débutant) :
Tout ça n’est qu’un ensemble de fichiers .html qu’on met dans le même dossier. On peut les ouvrir depuis un navigateur, même sans internet, et on peut les héberger n’importe où (comme sur Vercel ou GitHub Pages).
Idée clé (niveau geek) :
Le “RG-univers” est une collection de pages HTML statiques liées entre elles. / joue le rôle de router minimal en JS, /gr est un cas d’usage concret (EP FWB) avec intranet front-only et console de commande simulée.
1. Comprendre la page BROL 2.0 (page d’accueil)

BROL 2.0, c’est ton dashboard central. On peut le voir comme une télécommande qui ouvre tous tes sites, fichiers et univers sans se perdre.

Sur https://rg-ashy.vercel.app/, tu vois :

  • À gauche : une liste de modules (pages, jeux, intranets…)
  • Au centre : un viewer qui affiche la page choisie
  • À droite : des embeds rapides (YouTube, PDF, pingouins…)
Modules = pages / fichiers
Chaque module correspond à un fichier dans ton dossier (par ex. gr.html, intranet.html, etc.) ou à une vue spéciale.
// Dans BROL 2.0, en bas du fichier : const modules = [ { id: "gr", title: "Mouvement EP citoyenne", file: "gr.html", category: "intra", kind: "Portail & intranet EP", description: "Site sérieux pour l’éducation permanente FWB." }, // ... autres modules ];
Embeds = contenus externes
Les “embeds” permettent de charger un PDF, une vidéo, une image dans le viewer sans changer de page.
// Exemple d’embed dans BROL 2.0 : const embeds = { plaidoyer: { label: "PDF plaidoyer", url: "plaidoyer-jp.pdf", description: "Un exemple de document à analyser." }, // ... };
Version simple : BROL 2.0 ne “contient” pas tout. Il se contente d’ouvrir des fichiers qui sont dans le même dossier que lui. Si tu enlèves un fichier, le module ne marchera plus.
Version geek : BROL 2.0 est un SPA minimaliste sans framework. Tu ajoutes / retires des “routes” en modifiant le tableau modules. Le viewer est un <iframe> ou un conteneur qui change de src selon le module sélectionné, sans API backend.
2. Le site “GR” : mouvement d’éducation permanente

La page /gr montre ce qu’on peut faire de sérieux avec ce système : un portail d’éducation permanente citoyenne pour la Fédération Wallonie-Bruxelles, avec vitrine publique + intranet de démonstration.

Partie publique
On y trouve :
  • Une phrase d’ouverture (“un mouvement qui assume l’ambition de faire mieux”)
  • La mission, les axes, les publics, les territoires
  • Une section “Transparence & conformité” (statuts, rapports, EP FWB)
Ça donne une image crédible pour des partenaires, des pouvoirs publics, des financeurs.
Partie intranet / console
Vers la fin de la page, tu vois :
  • Un tableau de bord interne avec quelques indicateurs
  • Une maquette de formulaire d’action / animation
  • Un “registre des documents” et des bénévoles
  • Une console accessible par /console dans le texte
Dans la version statique, tout ça reste fictif, mais la structure est prête pour une vraie mise en production.
À retenir :
La page gr.html n’est “qu’un fichier HTML” dans le même dossier. Ce fichier peut être :
  • ouvert directement dans ton navigateur
  • appelé depuis BROL 2.0 comme un module
  • hébergé en ligne sur un service statique
Pour les curieux :
  • Le texte colle aux exigences typiques du décret EP FWB (mission, axes, publics, gouvernance).
  • La section intranet est un “mock” de back-office, utilisable comme cahier des charges futur.
  • La console mentionnée (/console – mouvement-ep-fwb) simule un CLI interne pour déployer des commandes symboliques (vision, statut, axes, etc.).
3. Tout mettre dans un dossier (version clé USB / ordi perso)

Le plus simple pour vraiment comprendre, c’est de te dire : “Tout tient dans un dossier sur mon ordinateur ou ma clé USB.”

Par exemple :

# Structure minimale sur ton ordinateur : rg-univers/ ├─ index.html # BROL 2.0 – Dashboard central ├─ gr.html # Mouvement d’éducation permanente (GR) ├─ rg-tutoriel.html # CE FICHIER : le guide pour tout comprendre ├─ plaidoyer-jp.pdf ├─ dystopie.pdf └─ autres-pages.html # intranet, jeux, etc. (optionnel)

Ensuite, tu :

  • ouvres index.html (BROL 2.0) dans ton navigateur
  • renommes les modules pour qu’ils pointent vers gr.html, rg-tutoriel.html, etc.
  • peux ouvrir rg-tutoriel.html à côté pour relire les explications
Hyper important :
Si un module dans BROL pointe vers "gr.html", alors le fichier gr.html doit être présent dans le même dossier.
Petite astuce geek :
Si tu héberges sur Vercel/GitHub Pages, index.html devient / et gr.html devient /gr. C’est exactement ce qui se passe sur rg-ashy.vercel.app.
4. Pas à pas : de “je ne comprends rien” à “j’ai tout pigé”

Voici une séquence simple à suivre, même si tu n’aimes pas la technique.

  1. Ouvrir ce guide → Tu lis cette page (rg-tutoriel.html). Tu retiens : il y a un dashboard (BROL) et un site sérieux (GR).
  2. Ouvrir BROL 2.0 → Sur ton ordi : index.html dans le même dossier. → En ligne : https://rg-ashy.vercel.app/.
  3. Cliquer sur le module “GR” → Tu vois la page “Mouvement d’éducation permanente citoyenne”. Tu la parcours une fois, sans chercher à tout analyser.
  4. Relire ce guide en parallèle → Tu reviens ici, tu lis chaque section en comparant avec ce que tu vois réellement dans le site.
  5. Ajouter un module perso → Tu crées une page simple (ex. mon-projet.html) → Tu l’ajoutes au tableau modules dans BROL 2.0. → Tu vois apparaître ton propre bouton dans la liste.
Si tu bloques :
Tu peux considérer chaque fichier HTML comme une feuille A4 : - tu la modifies - tu l’enregistres - tu la ré-ouvres dans le navigateur Rien ne “casse” tout l’ordinateur : au pire, la page s’affiche mal jusqu’à ce que tu corriges.
Si tu veux aller plus loin techniquement :
  • Utiliser un éditeur comme VS Code.
  • Mettre ton dossier complet dans un dépôt Git.
  • Déployer sur Vercel / GitHub Pages pour avoir exactement ton propre rg-*.
5. Et après ? (idées d’extension)

Une fois que tu as compris la logique BROL + GR + dossier, tu peux t’amuser à étendre sans tout casser.

Idées “grand public”
Sans toucher au JavaScript compliqué :
  • Modifier les textes du site GR pour coller à un vrai projet.
  • Ajouter une page “FAQ” ou “Nous contacter” comme nouveau module.
  • Créer une page “Pas à pas pour créer ton propre site EP”.
Idées “geek / avancé”
En restant toujours en HTML/CSS/JS statique :
  • Améliorer la console du site GR pour qu’elle lise un peu l’URL ou l’heure.
  • Ajouter des petits formulaires qui stockent temporairement des infos dans localStorage.
  • Faire un second dashboard BROL pour un autre univers (pro, perso…).

L’idée générale : rester sur du statique (HTML/JS/CSS, pas de base de données), pour que tout puisse tenir sur une clé USB, un dépôt Git, ou un hébergement ultra simple.
